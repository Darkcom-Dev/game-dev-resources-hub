Introduction

Key Points:

  • 3D models optimized for maximum performance on mobile devices and VR.
  • Texture atlas system that significantly reduces resource consumption.
  • Designed to maintain high visual quality with minimal performance impact.
  • Ideal for games requiring 90+ FPS.

Our products offer a specialized solution for developers seeking to balance visual quality and performance. Each model is meticulously optimized to function efficiently on mobile devices, VR systems, and low-resource hardware without compromising the visual quality of your project.

Texture Atlas System

Our texture atlas system provides multiple advantages:

  • Improved Performance: Significant reduction in draw calls.
  • Memory Efficiency: Optimized VRAM usage.
  • Flexibility: Individual manipulation of meshes while maintaining efficiency.
  • Simplicity: A single shared material reduces project complexity.

Ideal Use Cases

  • VR/AR: Applications requiring 90+ FPS for a smooth experience.
  • Mobile Games: Projects needing broad compatibility with devices.
  • E-Sports: Titles demanding high performance (100+ FPS).
  • WebGL: Browser-based 3D applications.

Important Note:

Each product includes a specific manual with detailed implementation and optimization instructions for its particular use case. This general manual provides an overview of our modeling system and best practices for its use.

Environment Setup

Required Software Installation

The product does not require additional installations as long as the minimum software requirements specified in the documentation are met. Ensure compatible versions are used to avoid compatibility issues.

Engine Configuration

While the engine can render models without special configurations, mobile game developers should consider the following recommendations to optimize performance.

Resolution Limitations

This product is optimized for resolutions of 1080p or lower. It is not recommended for use on 4K or 8K displays due to the performance impact unless the game camera is distant, such as in real-time strategy (RTS) games.

Graphic Optimizations

The following optional graphical optimizations are suggested to maximize efficiency without sacrificing visual quality:

  • Custom Shaders: Design shaders to combine grayscale textures such as Roughness, Ambient Occlusion (AO), or Translucency. This reduces the number of individual textures and optimizes memory usage.
  • Texture Compression: For performance-critical projects, consider merging AO with Albedo via color multiplication or storing Roughness in the Alpha channel. This may require baking textures or making irreversible adjustments.
  • Post-Processing: Use effects like SSAO (Screen Space Ambient Occlusion) to enhance lighting without relying on AO textures. Adding a Bloom effect can also improve realism.
  • Shader Optimization: If skilled in shader programming, combine multiple effects into a single pass to reduce graphical load.

Compression Formats

For textures, the recommended compression format is ASTC, as it provides high quality and compatibility with modern devices. If the target device encounters issues or glitches, switch to ETC2 as an alternative.

Model Installation

Model Installation Guide

Most game engines are compatible with files exported from Blender. However, some engines interpret axis orientation differently. By default, all models are configured with the Z-axis pointing up, but in certain engines, such as Unity, the Y-axis points up. To address this discrepancy, you can:

  • Rotate the models 90° on the Y-axis in Blender before exporting.
  • Use formats such as .glb or .fbx, which generally adjust this mismatch automatically.

Material Setup

Although engines are compatible with Blender, the fidelity of materials is often affected during import. Most engines assign a default PBR (Physically Based Rendering) material and approximately map the textures. To optimize materials:

  • The texture r-rough-b-ao.png combines roughness information in the red channel (Roughness) and ambient occlusion in the blue channel (Ambient Occlusion).
  • You may need to configure a custom shader to correctly interpret these data.
  • The additional files package includes a GLSL shader compatible with Blender, which can serve as a base for designing a custom shader in your preferred engine.

Texture Atlas Setup

Some engines, such as Unreal Engine, allow combining multiple textures into a large Texture Atlas of up to 8192x8192 pixels. If your engine supports this feature, you can use it to manage textures more efficiently. However:

  • Most engines do not support this feature automatically, which may require manual adjustments.
  • The models are already designed with an optimized Texture Atlas, so additional configuration is generally unnecessary.

Model Structure

Model Hierarchy

Each product is delivered as a single file in a specific format (.blend, .fbx, .obj, .glb, etc.), containing multiple optimized meshes. These meshes share a single texture to ensure efficiency and compatibility. Key features include:

  • Uniform Scale: All meshes have a scale of 1.0 on each axis.
  • Standard Rotation: Rotations are set to 0.0 to prevent inconsistencies during import.
  • Metric Units: Models are configured in meters to align with the International System of Units (SI).

If the model is designed to include animations, movable parts are separated from static parts to simplify manipulation. Most products do not include preconfigured bones or animations unless explicitly specified. For further details, consult the model-specific manual.

Texture Atlas Usage

Meshes utilize an optimized Texture Atlas to maximize texture area usage and minimize graphical load. The UV maps are configured with the following features:

  • Efficient Distribution: UV islands are proportioned to match the model size, ensuring uniform resolution across all meshes.
  • Minimal Stretching: UV cuts are designed to capture maximum detail without noticeable distortions.

his approach, however, has a limitation: separating meshes to apply individual textures can be challenging. In such cases, you may need to bake textures into a new UV map.

Optimización para Juegos Móviles

Consejos para Reducir el Impacto en el Rendimiento

Para optimizar el rendimiento en dispositivos móviles, se recomienda comprimir las texturas utilizando el formato ASTC, siempre que el dispositivo de destino sea compatible. En caso de incompatibilidad, opte por ETC2 como alternativa. Todos los productos están diseñados con un límite máximo de 5000 vértices por modelo para garantizar una carga gráfica eficiente. Aunque es posible reducir aún más la cantidad de polígonos, hacerlo podría comprometer la calidad visual de la malla.

Configuración de Modelos de Detalle (LOD)

En la mayoría de los casos, los modelos no están configurados con niveles de detalle (LOD, por sus siglas en inglés). Si decide implementarlos, la mejora de rendimiento puede ser marginal dependiendo del proyecto. Una alternativa eficiente es utilizar un shader basado en tesselación para suavizar los bordes según sea necesario, optimizando tanto la calidad visual como el rendimiento.

Iluminación y Sombras

Para proyectos con ciclos de día y noche o iluminación dinámica, se recomienda implementar shaders que calculen la iluminación a nivel de vértices y manejen sombras con un sesgo mínimo o sin sesgo. Esto reduce la carga computacional de la GPU.

Si el proyecto utiliza iluminación estática, aproveche el horneado de sombras y normales en los objetos estáticos. Esto mejora significativamente el rendimiento al eliminar cálculos de iluminación en tiempo real.

Manipulación y Uso del Modelo

Posicionamiento de Objetos

Las mallas están diseñadas con el pivote ubicado en la base del modelo para facilitar su posicionamiento sobre superficies planas. En el caso de objetos irregulares que puedan ser lanzados o manipulados físicamente, el pivote se encuentra en su centro de masa. Esto garantiza una manipulación más intuitiva en el motor de juego.

Además, las mallas están configuradas con dimensiones estandarizadas y proporcionales a los objetos reales, evitando la necesidad de realizar ajustes de escala en la mayoría de los casos. Esta modularidad simplifica su integración en diferentes escenarios.

Aplicación de Animaciones

Por defecto, la mayoría de los modelos no incluye animaciones predefinidas. En los casos en los que el modelo cuente con animaciones, encontrará instrucciones detalladas en el manual específico incluido con el producto.

Interacción con Scripts y Físicas del Motor

La mayoría de los modelos están diseñados como objetos estáticos, libres de jerarquías complejas, lo que simplifica su interacción con sistemas de físicas y scripts en el motor de juego. Sin embargo, algunos modelos pueden incluir características adicionales, como datos almacenados en los colores de los vértices. Cuando esto ocurra, se proporcionará en los archivos adicionales:

  • Ejemplos de uso mediante scripts específicos.
  • Shaders que aprovechan características como el movimiento por viento basado en colores de vértices.
  • Enlaces a demostraciones web para ilustrar su funcionalidad.

Resolución de Problemas

Los productos estan diseñados para evitar la mayor cantidad de problemas posible, pero si se presenta algun problema puede seguir esta guia rapida:

UVs mal asignadas

Si ese es el caso, habra Blender y en el apartado de mallas, observe cuantos mapas UV contiene, puede que por error se haya pasado mas de un mapa UV, simplemente elimine el que no funciona. sin embargo las tiendas revisan previamente que el contenido funcione correctamente antes de publicarse.

Texturas no se muestran correctamente

Las texturas por defecto se encuentran en el mismo nivel que el archivo principal, puede que usted accidentalmente haya movido el archivo a otra ubicacion. para solucionarlo, simplemente reasigne las texturas en esa nueva ubicacion dentro del material.

Perdida de calidad en compresion

Los productos tienen sus texturas en formato PNG sin compresion con el fin que el usuario haga uso de la compresion segun sus necesidades, pero en caso de tener problemas de compresion, haga uso de algun formato que permita la compresion sin perdida de calidad y busque valores razonables para la compresion.

Preguntas Frecuentes (FAQ)

Preguntas Frecuentes (FAQ)

¿Por qué los modelos usan un texture atlas en lugar de texturas individuales?

El texture atlas reduce significativamente los draw calls y optimiza el uso de memoria GPU. Esto mejora el rendimiento en dispositivos móviles y VR, permitiendo mantener altos FPS sin sacrificar la calidad visual.

¿Qué hago si el modelo aparece rotado al importarlo?

Este es un problema común debido a las diferentes configuraciones de ejes en los motores. Puede solucionarlo rotando el modelo 90 grados en el eje Y, o usar los formatos .glb/.fbx que suelen manejar mejor esta situación.

¿Cómo manejo las texturas que combinan múltiples datos en sus canales RGB?

Use un shader personalizado que separe los canales. Por ejemplo, para una textura r-rough-b-ao.png, extraiga la rugosidad del canal rojo y la oclusión ambiental del canal azul. En los archivos adicionales encontrará shaders GLSL de ejemplo.

¿Puedo modificar las UVs para usar texturas individuales?

Es posible, pero no recomendable. Requeriría rehacer el UV mapping y hornear nuevas texturas, lo que podría resultar en pérdida de calidad y eliminaría los beneficios de optimización del texture atlas.

¿Qué formato de compresión de textura debo usar?

Se recomienda ASTC para la mayoría de dispositivos móviles modernos. Si encuentra problemas de compatibilidad, use ETC2 como alternativa.

¿Por qué las texturas están limitadas a 4096x4096?

Este límite asegura compatibilidad con la mayoría de dispositivos móviles y motores, mientras mantiene un buen balance entre calidad visual y rendimiento. Texturas más grandes podrían causar problemas en equipos modestos.

¿Puedo usar estos modelos en juegos de alta resolución (4K/8K)?

No es recomendable para vistas cercanas en 4K/8K. Sin embargo, funcionan bien en 1080p o en vistas alejadas como en juegos RTS, incluso en resoluciones mayores.

¿Es posible reducir aún más los polígonos de los modelos?

Los modelos ya están optimizados con un límite de 5000 vértices. Una reducción adicional probablemente deterioraría visiblemente la calidad de la malla.
